Report using the Pyramid Principle. Answer top-down, not narration bottom-up.

## Three rules (always)

1. **Conclusion first** — Lead with the single most important answer/recommendation. Never bury it after context.
2. **Group by kind (MECE)** — Group supporting points so they don't overlap and don't leave gaps (Mutually Exclusive, Collectively Exhaustive). 3–5 groups max per level.
3. **Vertically provable** — Each point must survive a "why?" from the level above and be supported by the level below.

## Shape

```
Conclusion (the one answer)
├─ Point 1  ── evidence/reason
├─ Point 2  ── evidence/reason
└─ Point 3  ── evidence/reason
```

## Horizontal relation — pick one per group

- **Parallel** — independent angles all supporting the conclusion.
- **Deductive/sequential** — a reasoning chain where each point leads to the next.

## When to open with SCQA instead of a bare conclusion

Use when the reader needs framing (proposals, persuasion, status updates):
- **S**ituation → **C**omplication → **Q**uestion → **A**nswer.
- Variants: ASC (open with answer), CSA (lead with the worry), QSCA (lead with confidence).
For a quick factual answer, skip SCQA and just state the conclusion.

## When NOT to use the pyramid

Storytelling/emotion, talking about feelings, when you haven't thought it
through yet (use the pyramid as a thinking scaffold first, don't publish it),
or creative brainstorming.

## Self-check before sending

- Does the first sentence contain the actual answer? If not, rewrite.
- Remove any group that overlaps another (not ME) or any missing angle (not CE).
- Can each point answer "why should I believe this?" If not, add the supporting level or cut the point.
